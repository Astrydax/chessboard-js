ChessboardJS

A responsive mobile-first javascript chessboard library

Download now »

Main features

Easy integration
With just a few lines of code it is super easy to integrate this library to your project. Also as a bower package it is easy to install and manage the dependencies (bower install chessboard-js).
Responsiveness
The board is created in a way to resize itself automatically, so there is no need to call a resize function. This makes it possible to easily integrate ChessboardJS into responsive designs. You can easily check out this feature by looking at the examples and resizing the browser window.
Mobile-first
Apart from being responsive there is more for the mobile world. One thing is that the drag & drop feature - so common in many chessboard libraries - has been deliberately left out, as there is no much use for that on touch devices. No utilisation of the mouse-hover events either. Also as much as possible the library was tested on mobile devices (Android, iOS, Windows Phone and Firefox OS).
Examples and documentation
Detailed examples show you the library in action. You can also use the yuidoc command (after installing with npm install -g yuidocjs) to generate developer documentation from the source on your machine.

Some words about the examples

During development the excellent javascript chessboard library chessboard.js has helped and gave a lot of inspiration. Since it is widely used, both the programming interface and the documentation has similar structures so that one can more easily compare the usage and features. There are some differences however. Just to list a few:

  • No drag & drop support in ChessboardJS (due to mobile oriented design). No mouse-over events are utilised for the same reason.
  • ChessbordJS automatically resizes board to container div, so it is ideal to integrate into responsive designs.
  • ChessbordJS supports both fonts and images as pieces.
  • Chessboard orientation in ChessboardJS changes the rank and file markers but setting the board in a changed orientation is not effected (see examples).
  • ChessbordJS doesn't support off-table pieces.
  • When generating the documentation with yuidoc, note that some of the methods are marked 'deprecated'. These are methods that can be used so to be compatible with chessboard.js, but not the prefered way to use the library.
  • ChessbordJS main class is named Chessboard (while chessboard.js uses the form ChessBoard). This might be important when moving from one to the other.
  • Other minor differences (ie. animation etc.)
